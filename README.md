# Flow 自定义AI代码审核步骤（伪智能体版）

## 项目介绍
本项目是一个基于 **Flow** 和 **阿里云云效（Yunxiao）** 的自动化 AI 代码审核工具，旨在通过调用大模型（如 Qwen、DeepSeek 等），对 Git 合并请求（MR）中的代码变更进行自动审查，并在发现问题时自动生成评论提交到 MR 中。

**最新升级：** 本项目已升级为“超级智能体”，具备多语言架构师级评审能力，内置了针对 Java, C++, C, Go, Python, Node.js, Vue 等主流语言的深度评审逻辑，并解决了传统 AI 评审中的 Diff 混淆、定位不准等痛点。

###  核心功能

- **多语言架构师级评审**：自动识别文件语言，加载内置的“资深架构师”级提示词，涵盖 **架构设计（DDD/Clean Arch）**、**开发规范（阿里/Google）**、**性能调优**、**安全隐患** 等维度。
- **精准行号定位**：采用结构化 JSON 输出与上下文感知技术，彻底解决 AI 评审中“评论打错行”或“误判删除代码”的问题。
- **智能并发控制**：内置智能并发限流机制，在保障评审速度的同时，有效防止触发大模型 API 的 Rate Limit。
- **开箱即用**：无需繁琐配置，默认内置最佳实践 Prompt，同时也支持自定义扩展。

### 解决的问题
1. **人工评审效率低**：传统的代码评审依赖人工逐行检查，容易遗漏逻辑错误、安全隐患等问题。
2. **多语言栈挑战**：单一评审规则难以适配多语言项目，本工具内置多语言专家知识库，一站式解决。
3. **AI 幻觉与误判**：通过严格的上下文约束和 JSON 结构化输出，大幅降低 AI 的误报率。

## 如何使用？

### 1. 安装依赖
```bash
npm install --registry=https://registry.npmmirror.com
```

### 2. 配置参数
配置以下环境变量（在 step.yaml 中定义）：

| 参数名称          | 描述                                                         |
| ----------------- | ------------------------------------------------------------ |
| `modelName`       | 大模型名称（例如 qwen-max、deepseek-v3）                     |
| `yunxiaoToken`    | 云效 Token（需具备代码只读、MR读写权限）                     |
| `aiBaseUrl`       | 模型厂商 API 地址                                            |
| `dashscopeApikey` | 模型服务商 API Key                                           |
| `temperature`     | 模型温度值（建议 0.2 以保持输出稳定性）                      |
| `llmChatPrompt`   | **[新特性]** 自定义提示词。**强烈建议留空**，系统会自动根据文件类型加载内置的架构师级 Prompt。 |

### 3. 构建并运行
```bash
npm run build
node dist/index.js
```

### 4. 配置流水线

1. 安装 flow-cli：
   ```sh
   npm install -g @flow-step/flow-cli --registry=https://registry.npmmirror.com
   ```

2. 登录云效组织：
   ```sh
   flow-cli login
   ```

3. 发布步骤：
   ```sh
   flow-cli step publish --auto-version
   ```

4. 在云效流水线中使用该步骤，并配置相应的环境变量。

## 核心特性详解

### 1. 多语言架构师 Prompt
系统会根据文件扩展名自动加载对应的 Prompt，例如：
- **Java**: 关注 JVM 内存泄漏、线程安全、Spring 事务、阿里开发手册规范。
- **Go**: 关注 Goroutine 泄漏、Channel 死锁、Uber Go 规范。
- **C++**: 关注 智能指针、RAII、内存安全、Google C++ 规范。
- **Vue/React**: 关注 组件生命周期、Hooks 依赖、响应式陷阱。

### 2. 解决 Diff 混淆
通过在 Prompt 中注入强制指令：“只关注 diff 中以 '+' 开头的行”，并结合上下文分析，有效避免了将“删除的代码”误判为“新增代码”的问题。

### 3. 结构化输出
模型输出被强制约束为 JSON 格式，包含：
- `type`: 缺陷类型（如：逻辑错误、安全隐患、并发问题）
- `line`: 精确的 Diff 行号
- `content`: 问题详情（引用具体代码）
- `suggestion`: 修复建议

### 4. 架构优化
- **并发控制**：将默认并发数从 30 降至 5，并增加了重试机制，显著提升了在 API 限流场景下的稳定性。
- **错误处理**：增强了 JSON 解析的容错性，即使模型偶尔输出非标准 JSON，也能尝试回退处理。
